# PROTOTYPE_BRIEF_PLANNER_PRO.md

## Direction
Planner Pro

## Objective
Create the most effective working planning surface for active team use without drifting into PM software.

## Design thesis
The product should support real planning work, not just polished presentation. This direction explores the maximum useful density that still feels calm and marketing-native.

## Primary moment to optimize
A marketing manager is actively shaping and refining the annual plan with enough visible context to work quickly.

## Core emphasis
- stronger editing clarity
- richer filter visibility
- more obvious owner/status signals
- higher planning speed than Executive Canvas

## Required prototype screens
1. project setup
2. default year view
3. detail panel editing state
4. filtered planning state
5. overlap stack state
6. export/review mode

## Interaction priorities
- editing should feel fastest in this direction
- detail panel can be more prominent
- filters must be visible but not noisy
- quick add should support high-frequency use

## Visual rules
- medium density, not dashboard density
- metadata hierarchy must stay disciplined
- avoid enterprise table/Gantt feel
- status and ownership should be readable without taking over the card

## Success criteria
- users can make planning changes with less panel-hopping
- visible metadata improves speed without cluttering exports
- still clearly feels like campaign planning, not task management

## Failure signs
- screen starts reading like monday.com / smartsheet lite
- card density damages annual readability
- export mode requires major simplification to look acceptable
